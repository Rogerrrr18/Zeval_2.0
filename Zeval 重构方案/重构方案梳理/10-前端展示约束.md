# 10-前端展示约束

本文档约束 Zeval 重构后的前端页面行为，确保用户看到的是“可解释的结果与可执行的行动”，而不是“内部实现的解释”。

## 10.1 总体原则

1. 不暴露内部实现：不展示算法名、prompt stage、规则函数名、模块文件路径。
2. 不展示开关性质的“内部 toggle”：例如“启用 LLM 主观评估”这种 checkbox 必须从 UI 移除。
3. 主观与客观分离展示：用户能立刻看出哪些是统计指标、哪些是 LLM 主观评审。
4. 失败必须可见：LLM 主观指标不可用时不能假装显示出来，必须明确提示降级。
5. 证据可追溯：任何主观分数与建议必须能点击查看证据。
6. 中文优先：所有用户可见文案使用中文，避免与代码字段名混排。

## 10.2 页面级约束

### 10.2.1 工作台 `/workbench`

- 顶部进入入口：上传区、运行评估按钮、跳转“在线评测”按钮，不重复出现“运行评估”。
- 评估运行后展示：图表面板、指标摘要、建议、badcase、保存 baseline 入口。
- 页面状态在跳转“在线评测”后再返回时保留（基于 sessionStorage 或后端 run 持久化），避免上传数据丢失。
- 不展示：模块名、是否 LLM、规则降级原因等内部信息。
- 必须展示：主观指标的来源标识（仅“LLM 评审 / 规则降级 / 不可用”三类语义化标签）。

### 10.2.2 在线评测 `/online-eval`

- 输入：客户 ID、baseline 选择、回复 API 配置。
- 不展示“启用 LLM 主观评估”checkbox，主观评估始终启用。
- 输出：客观对比图、主观对比图、风险信号对比、建议差异。
- 失败必须显示：回放 turn 失败数量、超时次数。

### 10.2.3 案例池 `/datasets`

- 列表视角：goodcase / badcase 切换、标签筛选、近重复提示，**主筛选维度为"能力维度"**（multi_turn_coherence / tool_calling_correctness / ...，详见 [`15-能力维度评测与归因.md`](./15-能力维度评测与归因.md)）。
- 在能力筛选下，二级筛选支持 `source`（8 类来源）与 `failureLayer`（L0–L8）。
- 详情视角：transcript、来源 evaluation_run、绑定建议与证据、**能力维度标签 + 失败层标签 + 归因方法 + 归因置信度**。
- 不展示 `normalizedTranscriptHash` 原值，可显示"近似重复"徽章。
- `failureLayer` 不直接展示英文枚举值（如 `L5_tool_selection`），必须按词典翻译为中文（"工具选择层"），鼠标悬浮可显示英文 key 与简要说明。

### 10.2.4 调优包 `/remediation-packages`

- 列表与详情视角：issue-brief、spec、badcases、acceptance-gate 四个 artifact。
- 必须显示触发指标 key 的中文释义，不直接展示英文 key。

### 10.2.5 Chat `/chat`

- 不展示 prompt 内部、skill 内部 trace。
- 引用证据时使用 "指标名 + 证据片段" 形式，不显示 evidenceSpanId 字符串。
- 当用户描述 agent 痛点（如"多轮对话不行"）时，Copilot 必须按 `diagnose_capability_gap` skill 流程：先抽出能力维度候选并反问澄清，再拉对应桶的信号分布与归因结果，最后给优化路径建议。展示时使用中文能力名（"多轮一致性"），不显示 `multi_turn_coherence` 字面值。
- 归因结果展示必须包含三段："失败层（中文）+ 置信度 + 来源（规则/分类器/对照实验）"；不允许只显示结论不显示来源。

### 10.2.6 实验路由 `/experiments`（P5 阶段新增）

- 列表视角：按能力维度分组，显示自变量轴与候选数。
- 详情视角：展示候选 × 评测指标矩阵；高亮性价比最高的组合。
- 不展示模型 API key、内部 prompt 全文；可下载脱敏后的实验报告。

## 10.3 通用 UI 元素

| 元素 | 约束 |
| --- | --- |
| 主观分数卡片 | 必须显示数值 + 中文维度名 + 来源标签（LLM 评审 / 规则降级 / 不可用） |
| 建议卡片 | 必须显示问题、影响、建议动作三段；可选显示触发指标的中文名 |
| 图表 | 必须有标题、坐标轴单位、悬浮提示；不显示原始字段名 |
| 错误提示 | 中文化；区分“数据问题”“模型问题”“网络问题” |
| 加载状态 | 主观评估、回放、合成需要显示进度或阶段文字 |

## 10.4 用户文案策略

- 使用业务语言：例如"答非所问风险"而非 `offTopicOrIgnoreRisk`，UI 上对应文案必须在词典中维护。
- 能力维度（12 个）与 Harness 层（L0–L8）必须在 `i18n/capability.zh.json` 与 `i18n/harness-layer.zh.json` 中维护中文映射（如 `multi_turn_coherence → "多轮一致性"`，`L5_tool_selection → "工具选择层"`），UI 不直接拼接英文字面值。
- 失败文案：
  - LLM 不可用：“当前模型评审暂不可用，已展示规则结果”。
  - 数据缺失时间戳：“未识别到时间字段，部分时序图表不可用”。
  - 回放 API 失败：“回复 API 在 N 轮中超时 / 返回错误”。
- 不要在 UI 文案中展示模型名、prompt 版本号；这些信息可放在“运行详情”或运维侧。

## 10.5 状态持久化

- 工作台：上传文件名、解析结果、评估结果、对话快照在 sessionStorage 中持久化，跳转返回不丢失。
- 在线评测：客户 ID、回复 API 配置、最近一次对比结果在 sessionStorage 持久化。
- 不长期持久化敏感字段（例如 token），关闭页面即清除。

## 10.6 性能与可访问性

- 单次评估首屏加载控制在 5 秒内，长时间任务转入异步并显示阶段进度。
- 主图最多 6–8 张，避免一次性过多图表导致首屏过长。
- 中文字体使用系统字体或公共 CDN（如 Noto Sans SC），避免私有字体导致加载失败。
- 颜色对比度满足无障碍 4.5:1，关键状态颜色不仅依赖色相，也使用图标或文字。

## 10.7 必须移除的 UI 元素

- “启用 LLM 主观评估” checkbox。
- “使用降级规则计算” 显示。
- 任何带 `workspaceId` 字面值的输入或显示。
- 任何在前端展示 `metricKey` 英文名的位置（必须翻译）。

## 10.8 验收

1. 任何页面截图都能让非技术用户独立读懂指标结果。
2. 任何主观结果都能在 UI 上看到来源（评审 / 降级 / 不可用）。
3. 任何建议都能在 UI 上点击查看触发指标与证据。
4. 任何错误都能在 UI 上得到中文解释，并包含建议处理动作。
