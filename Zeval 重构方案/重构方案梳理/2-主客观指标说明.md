# 2-主客观指标说明

## 指标分层

| 层级 | 名称 | 代表变量 | 是否调用 LLM | 说明 |
| --- | --- | --- | --- | --- |
| L0 | Raw Layer | `sessionId`、`timestamp`、`role`、`content` | 否 | 用户上传或 API 传入的原始对话行 |
| L1 | Normalized Layer | `turnIndex`、`timestampMs`、`activeHour` | 否 | 按 session 排序并补齐时序字段 |
| L2 | Topic Layer | `topicSegmentId`、`topicSummary`、`topicConfidence` | 条件调用 | 规则切分为主，长间隔连续性可调用 LLM |
| L3 | Emotion Layer | `emotionBaseScore`、`emotionScore`、`emotionEvidence` | 条件调用 | LLM 给基准情绪，本地规则修正最终分 |
| L4 | Row Signals | `responseGapSec`、`isQuestion`、`isTopicSwitch` | 否 | 后续客观与主观指标的行级输入 |
| L5 | Objective Metrics | 轮次分布、响应间隔、活跃时段、主题切换率 | 否 | 可稳定复算的统计指标 |
| L6 | Implicit Signals | 兴趣下降、理解障碍、情绪恢复失败 | 否 | 基于行为与文本规则的风险推断 |
| L7 | Subjective Metrics | 共情、追问有效性、答非所问风险、说教风险 | 是 | LLM Judge 或规则降级产出结构化评审 |
| L8 | Report Layer | charts、summaryCards、suggestions | 否 | 面向前端展示和行动建议 |

## 客观指标

客观指标只依赖 raw/enriched 行，不依赖 LLM 输出。即使 LLM 不可用，客观指标也必须完整返回。

| 指标 | 输入 | 计算时机 | 输出 |
| --- | --- | --- | --- |
| `sessionDepthDistribution` | 每个 session 的最大 `turnIndex` | enrich 完成后 | 摘要、图表、建议 |
| `dropoffTurnDistribution` | `isDropoffTurn` | enrich 完成后 | 断点图 |
| `avgResponseGapSec` | `responseGapSec` | enrich 完成后 | 摘要、节奏建议 |
| `activeHourDistribution` | `activeHour` | enrich 完成后 | 活跃时段图 |
| `topicSwitchRate` | `topicSegmentId` | topic 切分后 | 主题跳转风险 |
| `userQuestionRate` | user 行 + `isQuestion` | enrich 完成后 | 追问与理解建议 |
| `avgUserMessageLength` | user 行 content 长度 | enrich 完成后 | 参与度判断 |
| `avgAssistantMessageLength` | assistant 行 content 长度 | enrich 完成后 | 说教/冗长风险 |
| `userMessageLengthTrend` | 用户消息长度序列 | enrich 完成后 | 兴趣变化辅助判断 |

## 主观指标

主观指标默认调用 LLM Judge，但必须具备结构化输出和降级状态。重构后主观指标的数据库记录必须包含 `dimension_key`、`score`、`reason`、`evidence_span_id`、`confidence`、`source`、`judge_run_id`。

| 指标 | 输入 | LLM 介入 | 降级行为 |
| --- | --- | --- | --- |
| 情绪曲线 | topic segment 情绪分 + 行级 turn | 分段情绪基准可调用 LLM | 使用规则情绪词和结构权重 |
| 共情程度 | assistant 文本、用户情绪、隐式信号 | session 级 Judge | 使用共情词命中率 |
| 追问有效性 | 用户问题、assistant 回复、goal completion | session 级 Judge | 使用问题率、重复问、解决态规则 |
| 答非所问风险 | topic switch、理解障碍、证据片段 | session 级 Judge | 使用主题跳转和困惑表达规则 |
| 说教风险 | assistant 长度、命令式表达、用户反馈 | session 级 Judge | 使用说教词命中率 |
| 目标达成 | 用户意图、最后状态、解决态表达 | 规则不清晰时调用 LLM | 标记 `unclear` 或规则判断 |
| 恢复轨迹 | 失败点、修复点、目标达成结果 | completed 轨迹可调用 LLM 归因 | 只输出规则识别的失败/恢复窗口 |

## LLM Judge 调用点

| Stage | 当前含义 | 输入粒度 | 输出 |
| --- | --- | --- | --- |
| `topic_continuity_review` | 长间隔后是否延续上一主题 | 相邻主题上下文 | `isContinuation`、reason、confidence |
| `segment_emotion_baseline` | 主题段情绪基准 | topic segment transcript | polarity、intensity、baseScore、evidence |
| `subjective_dimension_judge` | 主观四维评审 | session transcript + signals | dimensions[] |
| `goal_completion_judge` | 目标是否达成 | session transcript | status、score、evidence |
| `recovery_trace_strategy` | 恢复策略归因 | failure/recovery span | strategy、quality、evidence |

## 指标到能力维度的映射

每个客观/主观指标必须显式标注它主要服务于哪些 `capabilityDimension`（详见 [`15-能力维度评测与归因.md`](./15-能力维度评测与归因.md)）。一个指标可服务多个能力维度。这张映射决定了 Copilot 在被问到"我想优化能力 X"时，应当展示哪些指标桶。

| 指标 key（节选） | 主要服务能力维度 |
| --- | --- |
| `topicSwitchRate` | `multi_turn_coherence` / `instruction_following` |
| `userMessageLengthTrend` | `multi_turn_coherence` / `latency_efficiency` |
| `avgResponseGapSec` | `latency_efficiency` |
| `empathyScore` | `instruction_following` / `style_format` |
| `offTopicOrIgnoreRisk` | `instruction_following` / `multi_turn_coherence` |
| `preachinessRisk` | `style_format` |
| `goalCompletion` | `task_decomposition` / `workflow_execution` |
| `recoveryTrace` | `self_correction` |
| `emotionRecoveryFailureRisk` | `self_correction` / `instruction_following` |
| `understandingBarrierRisk` | `instruction_following` / `knowledge_grounding` |

完整映射表维护在 `指标与DAG/主客观指标依赖表.xlsx` 的 `capability_dimension` 列中（迁移 P3 阶段补齐）。

## 指标透明化要求

1. 所有指标进入 `metric_registry` 或等价注册表，禁止只在 UI 层临时计算。
2. 每个指标都需要固定 key、中文名、层级、计算函数、输入字段、输出位置、**服务的能力维度列表**。
3. LLM 指标必须保存 prompt version、model、latency、raw response 摘要和 parse status。
4. 规则降级不应伪装成 LLM 结果，必须明确 `source=fallback` 或 `source=rule`。
5. 批量上传多 session 时，隐式信号和主观维度必须区分 dataset 级、session 级、segment 级。
