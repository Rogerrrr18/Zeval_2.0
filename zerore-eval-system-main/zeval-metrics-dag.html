Zeval Metric Architecture

主客观评测指标与 DAG 依赖关系

本文件按当前代码实现梳理，不按早期规划臆测。核心链路是 raw chatlog -> normalized rows -> enriched rows -> objective / subjective metrics -> charts, suggestions, badcases, remediation, validation。

输入 / 中间层 客观指标 主观指标 隐式信号 下游产物

代码依据

当前评测主链路入口为 runEvaluatePipeline。它先做 enrich，再并行组织客观、主观、场景、扩展指标和下游产物。

  • src/pipeline/evaluateRun.ts：评测总编排
  • src/pipeline/objectiveMetrics.ts：客观指标聚合
  • src/pipeline/subjectiveMetrics.ts：主观指标聚合
  • src/pipeline/signals.ts：隐式风险信号
  • src/pipeline/goalCompletion.ts：目标达成评估
  • src/pipeline/recoveryTrace.ts：失败-恢复轨迹
  • src/pipeline/emotion.ts：segment 情绪评分
  • src/pipeline/metricRegistry.ts：统一 gate/registry
  • src/pipeline/extendedMetrics/index.ts：可选扩展指标
  • src/pipeline/badCases.ts：badcase 资产抽取

整体 DAG

高层依赖图如下。图里只画主边，具体字段级依赖见后面的指标表。

核心分层

1. 中间层

  • Raw: sessionId, timestamp, role, content
  • Normalized: turnIndex, timestampMs, activeHour
  • Enriched: topic, emotion, gap, question, switch, dropoff flags

2. 指标层

  • 客观指标：纯规则/聚合，稳定可回归
  • 主观指标：LLM judge + 规则降级
  • 隐式信号：可解释风险信号，服务 badcase 和调优包

3. 产物层

  • 图表与 summary cards
  • 优化建议 suggestions
  • badcase assets, remediation package, validation gates

客观指标

指标 类型 直接父节点 计算逻辑 主要下游
sessionDepthDistribution
会话深度分布		 objective sessionId, turnIndex 按 session 取最大轮次，分到 1-3, 4-8, 9+。 工作台概览、数据质量判断
dropoffTurnDistribution
流失轮次分布		 objective isDropoffTurn, turnIndex 统计最后一条 assistant 消息所在轮次。 dropoffDistribution 图表、流失定位
avgResponseGapSec
平均响应间隔		 objective timestampMs, responseGapSec 所有有效相邻消息间隔求均值。 summary、suggestions、metric registry、long gap badcase
topicSwitchRate
话题切换率		 objective topicSegmentId, sessionId 每个 session 的唯一 topic segment 数减一，再按 session 求均值。 summary、registry、场景 KPI、off-topic 风险
userQuestionRepeatRate
用户重复提问率		 objective role=user, isQuestion, content 用户问题归一化后统计重复问题占比。 场景 KPI、调优包 target metric、badcase tag
agentResolutionSignalRate
解决态信号覆盖率		 objective role=assistant, content session 内 assistant 是否命中“已处理/提交/补发/退款/工单号”等解决态表达。 registry、场景 KPI、目标达成辅助解释
escalationKeywordHitRate
升级关键词命中率		 objective content session 是否命中“转人工/投诉/主管/经理/工单”等升级表达。 场景 KPI、调优包 target metric、badcase tag
activeHourDistribution
活跃时段分布		 objective timestamp, activeHour, role 按小时和角色统计消息量。 activeHourDistribution 图表
userQuestionRate
用户提问率		 objective role=user, isQuestion 用户消息里问句占比。 suggestions、兴趣衰减/理解障碍辅助判断
avgUserMessageLength
用户平均消息长度		 objective role=user, content.length 用户消息长度均值。 兴趣衰减、情绪 length weight、用户投入度判断
userMessageLengthTrend
用户消息长度趋势		 objective role=user, content.length, message order 对用户消息长度序列做简单线性斜率。 兴趣衰减解释、体验走势判断
avgAssistantMessageLength
AI 平均消息长度		 objective role=assistant, content.length assistant 消息长度均值。 说教/压迫感辅助解释、情绪 length weight

主观指标与信号

指标 / 信号 类型 直接父节点 计算逻辑 主要下游
emotionCurve
情绪曲线		 subjective emotionScore, turnIndex, sessionId 逐行输出 0-100 情绪分。情绪分来自 segment 基准分 + 本地权重修正。 情绪图、summary、suggestions、recovery、badcase
emotionTurningPoints
情绪拐点		 subjective emotionScore 相邻差值 同 session 相邻行情绪变化绝对值 >= 12 时记为拐点。 情绪复盘、证据定位
共情程度 subjective assistant 内容、topic segment、implicit signals LLM 模式按 session judge；降级模式按“理解/明白/支持/陪你/辛苦/正常”等共情表达密度打 1-5 分。 summary、registry empathy、suggestions、调优包 target
答非所问/无视风险 subjective isTopicSwitch, understandingBarrierRisk, transcript LLM 模式按 transcript + signal judge；降级模式为 5 - topicSwitchRowRate*8 - barrierRisk*2 后 clamp。 registry offTopicRisk、badcase、调优包 target
说教感/压迫感 subjective assistant 内容 LLM 模式 judge；降级模式按“应该/必须/你要/一定要”等命令式表达比例扣分。 场景 KPI alias preachiness、策略建议
情绪恢复能力 subjective emotionScore, emotionRecoveryFailureRisk 低谷轮次、恢复轮次和恢复失败信号共同决定 1-5 分。 场景 KPI alias emotionRecovery、调优包 target
goalCompletions
目标达成		 subjective session transcript、解决态、放弃/升级/失败表达、重复提问、情绪回升 规则先判 achieved / partial / failed / unclear；unclear 且启用 LLM 时再由 LLM 抽取 intent 并终裁。 summary 目标达成率、registry、badcase、recovery、remediation
recoveryTraces
失败-恢复轨迹		 subjective emotionScore, isQuestion, isTopicSwitch, 困惑表达, goalCompletions 先找失败点，再在 4 轮窗口内找道歉/重述/澄清/情绪恢复/目标达成信号，给出 status 与 qualityScore。 summary、suggestions、badcase、调优包示例
interestDeclineRisk
兴趣衰减风险		 signal 前后段用户长度、响应间隔、提问率 后半段回复变短、间隔拉长、提问意愿下降时累加风险分。 suggestions、场景 KPI、风险解释
understandingBarrierRisk
理解障碍风险		 signal 困惑表达、重复问题、用户提问后 assistant topic switch 命中“什么意思/不懂/没明白”等表达、重复提问或问后偏题时累加风险。 off-topic 维度、suggestions、badcase
emotionRecoveryFailureRisk
情绪恢复失败风险		 signal emotionScore, isDropoffTurn 低情绪后 4 轮内未恢复到 60 分，或断点前仍处低谷时升高风险。 情绪恢复能力、suggestions、badcase

情绪评分子 DAG

节点 父节点 作用
emotionBaseScore topic segment transcript；LLM 或规则关键词 每个 topic segment 的 0-100 基准情绪分。
emotionValenceWeight 正向/负向关键词命中数 按 ((positiveCount-negativeCount)/rows)*12 修正，范围 -8 到 8。
emotionLengthWeight 用户平均长度、assistant 平均长度 用户表达充分略加分，过短扣分；assistant 过长或过短扣分。
emotionStyleWeight 共情命中数、说教命中数 empathyCount*1.5 - preachyCount*3，范围 -8 到 8。
emotionGapWeight segment 内平均时间间隔 快响应小幅加分，45s/90s 以上扣分。
emotionRecoveryWeight 首尾用户情绪、assistant 支持表达 首负尾正或 assistant 支持会加分；负向且无支持扣分。
emotionRiskPenalty 最后角色、负向高强度、用户短负向 高风险 segment 的额外惩罚项。
emotionScore 以上所有节点 base + valence + length + style + gap + recovery - riskPenalty，clamp 到 0-100。

统一 Registry 与 Gate

metricRegistry 会把部分指标转换为 0-1 的 gate score，用于统一通过率、失败原因和场景模板映射。

Registry 指标 来源 归一化方式 默认阈值
avgResponseGapSec 客观 avgResponseGapSec 1 - avgResponseGapSec / 120 0.78
topicSwitchRate 客观 topicSwitchRate 1 - topicSwitchRate / 3 0.72
agentResolutionSignalRate 客观 agentResolutionSignalRate 直接使用 0-1 rate 0.68
goalCompletion 主观 goalCompletions session score / 5 后求均值 0.70
empathy 主观维度“共情程度” dimension.score / 5 0.68
offTopicRisk 主观维度“答非所问/无视风险” dimension.score / 5 0.68
serviceCallGrounding 结构化标注 service_call 参数可追溯到 dialogue state 的比例 0.85
serviceResultAvailability 结构化标注 service_call 有同 turn/service 结果的比例 0.95
schemaSlotCompliance 结构化标注 schema slot 覆盖率 0.95
traceStepEfficiency 真实 agent trace 完成 span 比例 - warning penalty 0.72
syntheticCoverage scenario synthetic seeds seedCount / 5 0.50

可选扩展指标

这些指标不会在普通客服日志里自动出现，只有当 API 请求提供 extendedInputs 时才会输出。

指标 输入 评估方式 默认阈值
extended faithfulness 回复忠实度 retrievalContexts LLM judge；降级为 query/response 关键词重合 0.70
extended hallucination 幻觉安全度 retrievalContexts LLM judge；分数越高越无幻觉 0.50
extended answerRelevancy 回复相关性 retrievalContexts LLM judge 或 query/response 关键词重合 0.70
extended contextualRelevancy 检索相关性 retrievalContexts LLM judge 或 query/context 关键词重合 0.70
extended toolCorrectness 工具调用正确率 toolCalls 工具名 0.5 + 参数 0.3 + 执行成功 0.2 0.80
extended knowledgeRetention 知识保持率 retentionFacts, optionally retrievalContexts LLM judge 或事实片段是否在后续 response 出现 0.70
extended toxicity 无毒度 retrievalContexts.response 规则关键词先扫，必要时 LLM 二次校验 0.50
extended bias 无偏见度 retrievalContexts.response 规则模式先扫，必要时 LLM 二次校验 0.50
extended roleAdherence 角色一致性 roleProfile, retrievalContexts 需要 LLM judge；未启用 LLM 时 skipped 0.70
extended taskCompletion 任务完成度 toolCalls 或 retrievalContexts LLM 60% + 工具成功率 40%；降级为工具成功率 0.70

下游产物依赖

Charts / Summary / Suggestions
产物 依赖 说明
charts.emotionCurve enrichedRows.emotionScore 按 session + turnIndex 画情绪曲线。
charts.dropoffDistribution isDropoffTurn, turnIndex 定位对话在哪些轮次停止。
charts.activeHourDistribution activeHour, role 看用户/assistant 在各时段消息量。
charts.topicSwitchFrequency isTopicSwitch, sessionId 每个 session 的切换次数。
summaryCards 客观指标 + 主观指标 + badcaseCount + optional structured metrics 产品首页指标卡片。
suggestions 低情绪轮次、平均情绪、目标达成率、响应间隔、topic switch、风险信号 输出 P0/P1/P2 优化建议。
Badcase / Remediation / Validation
节点 依赖 生成逻辑
badCaseAssets goalCompletions, recoveryTraces, repeated questions, escalation keywords, emotion drop, topic shift, long gap 按 session 抽取 tags、severityScore、evidence、transcript。
severityScore badcase tags goal_failed +0.34，recovery_failed +0.24，escalation +0.22，question_repeat +0.18 等。
remediation target metrics badcase tags + objectiveMetrics + subjectiveMetrics 把失败模式转为目标：提升目标达成/恢复/共情，降低重复提问/升级/响应间隔。
validation gates target metrics + replay/offline evaluation 对比 baseline/current，判断修复是否达到 acceptance gate。

目前 DAG 的工程含义

当前 Zeval 的真正核心不是“打几个分”，而是把同一批对话先变成可追溯的 enriched rows，然后所有客观指标、主观 judge、风险信号、badcase 和调优包都从这个中间层派生。后续做生产化时，应优先固定 enriched schema 和 metric registry version，否则评测结果很难长期可比。
