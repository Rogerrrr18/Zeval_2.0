一、研究背景与核心痛点

zerore 从真实历史对话挖掘 session 作为评测单元，更贴近生产，但带来两类问题：（1）固定多轮 user query 回放与优化后 Agent 的对话路径错位；（2）SimUser + LLM Judge 易递归、评判方差大、整体不可复现。

五、两种可行的历史数据回放评测方法

共同原则：保留历史真实性、允许 Agent 改路径、追求确定性或低方差评测。

方法一：状态锚定的因果回放

以状态转换链为锚；SimUser 由意图生成 query；支持提前达成跳过锚点；因果偏离判重错。可复现依赖：低温度 SimUser、rubric + 投票裁判、锚点标注规范、固定种子。

六、方法对比与实施建议

建议路径：先 1–2 周落地方法二（50–100 session 试点，校准意图提取与 rubric），再在工具型 session 上叠加方法一状态锚与因果检查；全程用在线 A/B 或影子流量验证离线指标与线上一致性。

局限提示：方法二依赖意图抽取质量；topic 跳跃时序列可能不准；追问模板需上限与终止条件，避免 SimUser 与 Agent 递归拉长。这些应在试点中量化（裁判 vs 人工一致率、平均轮次、提前完成率）。
