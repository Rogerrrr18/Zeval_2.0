4. 预定义 Schema · 一次调用双输出

抽取层须遵守事先写死的 JSON Schema（如 extract_session.schema.json + schema_version）。模型只做字段填空，保证：① 可复现（同 transcript、同 schema、同 seed 可校验/ diff）；② 任务驱动 query 不再重复执行——通过「触发条件 + 回填参考」把历史中已闭环的事实固化为待测 Agent 的 system 前缀，评测关注对话策略而非再次调外部系统。

4.1 业务概念 ↔ 工程键名

概念（中文） 工程键名 说明
意图序号 / 意图内容 / 轮次长度 intent_index · intent_text · turn_span_user_turns 轮次长度优先作为 §5 的 n_i
回填序号 / 触发条件 / 回填参考 refill_index · trigger_condition · refill_reference 可派生 injection_text；可选 bind_intent_index

5. SimUser：意图指针、计数与跳转

SimUser 负责生成 user，并依据上一轮 assistant 全文与当前意图 rubric 做 Judge（三分类：SATISFIED / NOT_SATISFIED / DEVIATION）。每完成「一条 user + 一条 assistant + 一次判定」记为一评测轮次，当前意图计数 c_i 加 1。

变量 含义
i 当前意图下标
c_i 当前意图已消耗评测轮次
n_i 历史中该意图 span 内 user 轮次数（默认算法见 MD；缺失则启发式并打标）
B_i ceil(α·n_i)，默认 α=2；n_i=0 时用 B_min=3；全局硬顶 B_global_cap 防爆炸

6. 可回填信息注入

将 refillables 拼为待测 Agent 的 system/developer 前缀，声明为「已知事实、勿重复执行外部查询」。与 SimUser 的 user 通道分离。默认每轮附带完整块；过长则首轮全文 + 后续摘要（见 MD）。

7. 指标 · 雷达基线 · SiliconFlow

指标 权重（报告） 备注
意图完成率 50% 达成数 / k
追问效率 20% 追问次数 / k，越少越好
偏离率 20% DEVIATION 轮次 / 总轮次（仅三分类）
轮次效率 10% T_eval / T_hist，<1 更省

基线 B：原始 CSV transcript 一次 Zeval evaluate（或等价）得到与雷达轴对齐的向量。动态 D：本实验四项 + 可选对动态 transcript 再评。首期不跑固定 user 第二臂。

SiliconFlow（与 zerore .env.example 一致）： ZEVAL_JUDGE_BASE_URL、ZEVAL_JUDGE_MODEL、ZEVAL_JUDGE_ENABLE_THINKING=false；新增 ZEVAL_INTENT_EXPERIMENT_API_KEY（空则回退 ZEVAL_JUDGE_API_KEY）。前端保存密钥后由后端写 env — POC 可用，生产需密钥管理服务（MD §7.5）。
